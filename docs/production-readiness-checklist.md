# 琥珀映画生产化预备清单

## 1. Admin API 第一版

- 管理员认证：独立管理员账号、角色、权限，不复用用户端 token。
- 套餐管理：新增、编辑、上下架、价格调整、封面图绑定。
- 门店管理：新增、编辑、营业时间、门店标签、可服务套餐绑定。
- 档期管理：批量生成档期、关闭档期、调整容量、查看预约占用。
- 订单管理：查询订单、改约、取消、完成、退款标记，所有敏感操作写审计日志。
- 底片管理：上传底片、绑定订单、设置可见状态、生成下载链接。

## 2. 对象存储接入

- 开发环境优先使用 MinIO，生产环境按部署条件选择 COS、OSS 或 S3 兼容服务。
- 数据库只保存文件元数据和 object key，不保存图片二进制。
- 原片、精修片、套餐封面、门店封面使用不同 object key 前缀。
- 用户侧下载使用短期签名 URL，避免永久公开原图。
- 上传完成后由后端落库，不能只以前端上传成功作为业务完成。

## 3. MySQL 与 Redis 迁移

- MySQL 使用 Flyway 迁移脚本初始化结构，不手工改生产库。
- 在上线前使用 MySQL 8 单独跑一次完整迁移和主链路接口测试。
- Redis 用于短信验证码、登录会话刷新、接口限流和热点档期缓存。
- H2 只作为本地开发数据库，不能作为生产数据库。

## 4. 发布前验证

- 后端：认证、实名、服务、门店、档期、预约、订单、底片接口测试通过。
- 前端：服务浏览、手机号登录、实名、预约提交、订单支付、底片查看主流程通过。
- 配置：生产环境密钥全部来自环境变量或密钥系统。
- 运维：日志包含接口路径、用户 ID、订单 ID 和错误信息，关键接口可追踪。

## 5. 已落地的生产化防呆

- 生产 profile 下禁止使用 H2 数据库和 H2 Driver。
- 生产 profile 下强制关闭 H2 console。
- 生产 profile 下禁止默认 `AMBER_TOKEN_SECRET` 和 `AMBER_ADMIN_TOKEN`。
- 生产 profile 下禁止 CORS 来源继续使用 `*`。
- 接口响应统一返回 `X-Trace-Id`，日志 pattern 会输出当前 traceId。

## 6. 仍需外部资源后才能执行

- 接入真实 MinIO/COS/OSS，并把 mock-local 上传凭证替换为真实短期签名。
- 接入短信平台，替换当前任意验证码登录。
- 接入微信登录、微信支付和支付回调验签。
- 使用 MySQL 8 独立环境执行完整 Flyway 迁移和主链路接口测试。
